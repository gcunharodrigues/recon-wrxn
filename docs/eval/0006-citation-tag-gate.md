## The bands (AC1–AC4)

- **resolved** — fact-verified: the target node provably exists AND, for an
  EVIDENCED_BY commit watermark, the sha is present in git history (R3 fold). The
  only band the `verified: true` view keeps.
- **inferred** — the link is emitted (target exists) but its commit watermark is
  unverified: well-formed but absent from history, or malformed. KEPT + flagged so
  the citation stays visible, never silently dropped; excluded by `verified: true`.
- **no edge** — precision-first: an evidence.symbols entry that does not resolve to a
  real node yields NO DOCUMENTED_BY edge at all — the resolver never mints a false `resolved`.
